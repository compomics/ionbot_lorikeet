      Target peptide database

      A limited set of the most widely studied proteomes can be selected to search the MS2 spectra against. These were obtained from the UniProtKB knowledgebase and processed with an silico tryptic digest, that allows for two missed cleavages, to create the target peptide database. The decoy peptide database was constructed similarly from the reversed proteins.The option to upload your own proteomes will be made available soon.

      MS2 spectrum files

      MS2 spectrum files should be in the Mascot MGF format. We recommend converting your RAW files with the freely available msconvert in the ProteoWizard library. Only MS2 peak picking needs to be selected, no other pre-processing should be performed.

      For protein inference to make sense, all spectra obtained from one sample should be in one spectrum file. Ionbot analyses each spectrum file separately.

      We recommend uploading spectrum files as a compressed package using Zip.

      Search settings

      The ionbot search requires you to set the following settings:

      Project name

      This name groups your spectrum files and can be used to access the ionbot search results.

      Protein database

      The target peptide database to search against.

      Enzyme

      The in silico digest pattern. Currently, only cleaving after lysine or arginine (K|R) is supported.

      Fragmentation method

      The MS2 fragmentation method applied to generate the spectra. Currently, only HCD is supported.

      MS Precursor error tolerance (ppm)

      The MS1 Precursor error tolerance in ppm. We recommend leaving this to the default 20ppm.

      Modifications as fixed

      A list of known fixed protein modifications.

      Modifications as variable

      A list of expected (variable) modifications. A maximum of four different modifications can be selected. Each PSM can have at most two expected modifications for the list, not including the fixed modifications.

      Enable Extended Modification Search

      If selected, an open modification search is performed that can match peptides that, next to the fixed and expected modifications,has one more unexpected modification selected from a list of known modifications, or has a mutation, or is semi-tryptic at the N-terminal.

      Result files

      The PSM result files contain all first-ranked and co-eluting lower ranked PSMs that passed the 5% FDR threshold computed using a reversed decoy database.To select a more confident set of PSMs the ‘q-value’ or ‘PEP’ columns (see below) can be used for further filtering based on statistical confidence. For PSMs, three results files are written:

      • ionbot.first.csv: contains all first-ranked PSMs
      • ionbot.lower.csv: contains all lower ranked PSMs with an edit distance of 3 or more to the first-ranked PSM
      • ionbot.lower.similar.csv: contains all lower ranked PSMs with an edit distance of 1 or 2 to the first-ranked PSM

      We refer to the ionbot paper for a detailed description of the lower ranked PSMs.

      Additional PSM information is written to an ‘ionbot.features.csv’ result file (see below).

      Proteins inference is performed using the ‘best-scoring-peptide-for-each-protein’ implemented in Percolator 3.0 1. This approach groups similar proteins inprotein groups to reduce the effect of peptides shared between proteins. The protein result files contain all protein groups that passed the 1% protein FDR threshold. Three protein result files are written:

      • ionbot.first.proteins.csv: contains all protein groups computed from the PSMs in ‘ionbot.first.csv’ that passed the 1% FDR threshold
      • ionbot.coeluting.proteins.csv: contains all protein groups computed from the PSMs in ‘ionbot.first.csv’ and ‘ionbot.lower.csv’ that passed the 1% FDR threshold
      The columns in a PSM result file are:

      ionbot_match_id

      An ionbot PSM identifier used to link the different result files together.

      spectrum_title

      The MS2 spectrum title (identifier) in the spectrum file.

      precursor_mass

      The observed MS1 precursor mass.

      peptide_mass

      The computed peptide match.

      observed_rentention_time

      The observed LC retention time

      charge

      The matched charge state.

      database_peptide

      The peptide sequence as it was found in the target database.

      matched_peptide

      The actual peptide sequence.

      modifications

      A list of sequence modifications that creates the matched peptidoform (matched_peptide) from the peptide in database_peptide. Each modification is separated by ‘|’ and is represented as X|Y, i.e. two modifications are written as X|Y|X|Y.

      Y can be a fixed, expected, or unexpected modification with X the position of the modification in the peptide sequence. Positions start at 1 with 0 and -1 indicating a N-terminal and C-terminal modification respectively.

      If the modification was not observed in MS2, then X is set to ‘x’. If Y is set to ‘semi-triptic, then X is the number of N-terminal amino acids that are removed to form the semi-tryptic peptide match.

      corrected_retention_time

      The peptide grouping corrected observed LC retention time. This value is used to compute RT-pred-error feature in the ionbot PSM scoring function.

      unexpected_modification

      The unexpected modification in the PSM, if any.

      database

      This is ‘T’ for a match against the target database and ‘D’ for a match against the decoy database.

      psm_score

      The ionbot PSM score. These are the distances to the hyperplane computed by the Support Vector Machine method.

      q-value

      The PSM q-value computed from the PSM scores.

      PEP

      The PSM posterior error probability computed from the PSM scores.

      proteins

      The list of proteins that match the peptide in database_peptide, separated by ‘||’.

      The columns in the features result file are:

      ionbot_match_id

      An ionbot PSM identifier used to link the different result files together.

      by-count

      The number of b- and y-ions matched.

      all-count

      The number of X-ions matched with X in [a,b,b-H2O,b-NH3,c,x,y,y-H2O,z], considering charge 1+ and 2+.

      by-explained

      The intensity explained by the b- and y-ions matched.

      all-explained

      The intensity explained by the X-ions matched with X in [a,b,b-H2O,b-NH3,c,x,y,y-H2O,z], considering charge 1+ and 2+.

      by-intensity-pattern-correlation

      The Pearson correlation between the observed and MS2PIP predicted b- and y-ion peak intensities.

      predicted_retention_time

      The DeepLC calibrated retention time prediction.

      rt-pred-error

      The absolute value of the difference between the predicted and the corrected observed retention time.

      The columns in a protein result file are:

      protein_group

      Protein groups are named after the proteins it contains (protein names are separated by ‘__’).

      protein_group_q-value

      The protein group q-value computed from the best PSM score for each protein group.

      protein_group_PEP

      The protein group posterior error probability computed from the best PSM score for each protein group.

      protein

      The protein name.

      position_in_protein

      The location of the peptide match in the protein (position of first amino acid in protein is 1).

      ionbot_match_id

      An ionbot PSM identifier used to link the different result files together.

      is_shared_peptide

      This is ‘True’ if the matched peptide is shared between different protein groups, ‘False’ otherwise.

      protein_length

      The length of the protein.

      protein_description

      The protein description.
